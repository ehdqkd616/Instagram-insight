## 1. 프로젝트 개요

### 1.1 프로젝트 목적

인스타그램에서 공식적으로 제공하지 않는 상세 활동 이력 — 팔로워/언팔 날짜, 계정별 좋아요·댓글 내역 — 을 사용자의 **공식 데이터 내보내기 파일**을 파싱하여 웹 UI로 제공하는 **로컬 분석 도구**입니다.

### 1.2 핵심 기능 요약

| # | 기능명 | 설명 | 주요 데이터 소스 |
|---|--------|------|----------------|
| F-01 | 팔로워 목록 조회 | 나를 팔로우한 계정 목록 + 팔로우 날짜 | `followers_1.json` |
| F-02 | 언팔 감지 조회 | 팔로잉 중이나 나를 팔로우 안 하는 계정 | `following.json` |
| F-03 | 계정 활동 검색 (수신) | 특정 계정이 내 게시물에 남긴 좋아요·댓글 | `likes.json`, `comments.json` |
| F-04 | 계정 활동 검색 (발신) | 내가 특정 계정에 남긴 좋아요·댓글 | `likes.json`, `comments.json` |

### 1.3 데이터 소스 — Instagram 데이터 내보내기

> **다운로드 방법:** 인스타그램 앱 → 설정 → 계정 센터 → 내 정보 및 권한 → 정보 다운로드 → **JSON 형식**으로 요청 (약 1~3일 소요)

| 파일명 | 위치 (압축 해제 후) | 포함 데이터 |
|--------|-------------------|-------------|
| `followers_1.json` | `connections/followers_and_following/` | 나를 팔로우한 계정, 팔로우 타임스탬프 |
| `following.json` | `connections/followers_and_following/` | 내가 팔로우하는 계정, 팔로우 타임스탬프 |
| `liked_posts.json` | `likes/` | 내가 좋아요 누른 게시물 (계정명, URL, 시간) |
| `liked_comments.json` | `likes/` | 내가 좋아요 누른 댓글 |
| `post_comments_1.json` | `comments/` | 내가 단 댓글 (계정명, 내용, 시간) |

> ⚠️ **한계 및 유의사항**
> - 실시간 데이터가 아닙니다 — 다운로드 시점의 스냅샷입니다.
> - '언팔'은 직접 기록되지 않고, `followers` / `following` 파일을 교차 분석하여 추론합니다.
> - 언팔 날짜는 정확하지 않습니다 — 마지막으로 팔로우한 날짜로 추정 표시됩니다.
> - 게시물 좋아요 데이터는 최근 수백 건까지만 포함될 수 있습니다.

## 2. 기능 상세 설계

### 2.1 F-01: 팔로워 목록 조회

#### 기능 설명
`followers_1.json`을 파싱하여 나를 팔로우하는 계정과 팔로우 시작 날짜를 테이블로 표시합니다.

#### UI 구성
- 상단: 총 팔로워 수 카드
- 검색창: 계정명으로 실시간 필터링
- 정렬 옵션: 최신 팔로우 순 / 오래된 순 / 계정명 가나다 순
- 날짜 범위 필터: 시작일 ~ 종료일 선택
- 결과 테이블: 계정명 / 팔로우 날짜 / 인스타그램 프로필 링크
- CSV 내보내기 버튼

#### 데이터 파싱 로직

`followers_1.json` 구조 예시:
```json
[
  {
    "title": "",
    "media_list_data": [],
    "string_list_data": [
      {
        "href": "https://www.instagram.com/username",
        "value": "username",
        "timestamp": 1710000000
      }
    ]
  }
]
```

| JSON 필드 | 설명 | 표시 형태 |
|-----------|------|-----------|
| `string_list_data[0].value` | 계정 아이디 (username) | 텍스트 |
| `string_list_data[0].href` | 프로필 URL | 클릭 링크 |
| `string_list_data[0].timestamp` | 팔로우한 UNIX 타임스탬프 | `YYYY-MM-DD HH:MM` |

---

### 2.2 F-02: 언팔 감지 조회

#### 기능 설명
`followers_1.json`과 `following.json`을 교차 분석하여, 내가 팔로우하지만 나를 팔로우하지 않는 계정(맞팔 안 됨)과, 내가 팔로우하다가 언팔당한 계정을 표시합니다.

#### UI 구성
- 탭 구분: **[나만 팔로우]** / **[맞팔 안 됨]** / **[내가 언팔당한 계정]**
- 각 탭: 계정 목록, 마지막 팔로우 날짜, 프로필 링크
- 검색 및 CSV 내보내기

#### 언팔 추론 로직

> 인스타그램은 '언팔 날짜'를 저장하지 않습니다. 따라서 아래 로직으로 추론합니다.

| 케이스 | 판별 조건 | 표시 내용 |
|--------|-----------|-----------|
| 맞팔 안 됨 | `following` 목록에 있으나 `followers` 목록에 없음 | 내가 팔로우 / 상대방은 안 함 |
| 나를 언팔한 계정 | 이전 `followers`에 있었으나 현재 없음 (비교 시) | 마지막 팔로우 날짜 추정 표시 |
| 내가 언팔한 계정 | 이전 `following`에 있었으나 현재 없음 (비교 시) | 마지막 팔로잉 날짜 추정 표시 |

> 💡 **히스토리 비교 기능**
> 두 시점의 데이터 내보내기 파일을 업로드하면 그 사이에 언팔한/당한 계정을 비교할 수 있습니다.
> 예: 1월 데이터 vs 3월 데이터 → 2개월 동안의 언팔 변동 추적

---

### 2.3 F-03 / F-04: 계정 활동 검색

#### 기능 설명
계정명을 검색하면, 그 계정이 내 게시물에 남긴 모든 좋아요·댓글 **(F-03)** 또는 내가 그 계정에 남긴 좋아요·댓글 **(F-04)** 을 한 화면에 통합 조회합니다.

#### UI 구성
- 상단: 계정명 입력 검색창 + **[내 활동 보기 / 상대 활동 보기]** 토글
- 결과 카드: 활동 유형 아이콘(❤️ 좋아요 / 💬 댓글) + 날짜 + 게시물 링크 + 댓글 내용
- 타임라인 뷰: 날짜순으로 정렬된 활동 피드
- 활동 유형 필터: [전체] / [좋아요만] / [댓글만]
- 날짜 범위 필터

#### 데이터 매핑

| 기능 | 소스 파일 | 검색 대상 필드 | 표시 정보 |
|------|-----------|---------------|-----------|
| F-03 (수신) | `liked_posts.json` | `title` 필드에서 계정명 포함 여부 | 게시물 URL, 좋아요 날짜 |
| F-03 (수신) | `post_comments_1.json` | 댓글 대상 게시물의 계정명 | 댓글 내용, 날짜, 게시물 URL |
| F-04 (발신) | `liked_posts.json` | URL에서 계정명 추출 | 게시물 URL, 좋아요 날짜 |
| F-04 (발신) | `post_comments_1.json` | 댓글 달린 계정명(URL 파싱) | 댓글 내용, 날짜 |

> **URL 파싱 예:** `https://www.instagram.com/p/AbCdEfG/` → 게시물 ID로 계정명 추출 (`liked_posts`의 `title` 필드 활용)

## 3. 시스템 아키텍처

### 3.1 전체 구조

로컬 실행 기반의 단일 서버 구조입니다. 데이터는 외부로 전송되지 않으며, 사용자 PC에서만 처리됩니다.

```
┌──────────────────────────────────────────────────────┐
│                  브라우저 (Web UI)                    │
│         HTML + CSS (Bootstrap) + JavaScript           │
└─────────────────────┬────────────────────────────────┘
                      │  HTTP (localhost:5000)
┌─────────────────────▼────────────────────────────────┐
│             Flask 웹 서버 (Python)                    │
│   Routes / API Endpoints / Template Rendering         │
└─────────────────────┬────────────────────────────────┘
                      │
┌─────────────────────▼────────────────────────────────┐
│             Parser / Service Layer                    │
│   follower_service.py / activity_service.py           │
└─────────────────────┬────────────────────────────────┘
                      │
┌─────────────────────▼────────────────────────────────┐
│        Instagram JSON 데이터 파일 (로컬)              │
│   followers_1.json / following.json / likes.json      │
└──────────────────────────────────────────────────────┘
```


## 4. API 엔드포인트 설계

모든 응답은 JSON 또는 HTML 렌더링 형식입니다.

| Method | Endpoint | 설명 | 주요 파라미터 |
|--------|----------|------|--------------|
| `GET` | `/` | 메인 페이지 (파일 업로드 화면) | — |
| `POST` | `/upload` | ZIP 또는 JSON 파일 업로드 처리 | `files` (multipart) |
| `GET` | `/followers` | 팔로워 목록 조회 페이지 | `sort`, `search`, `from_date`, `to_date` |
| `GET` | `/unfollowers` | 언팔 목록 조회 페이지 | `tab` (mutual/notfollowing) |
| `GET` | `/activity/search` | 계정 활동 검색 페이지 | `username`, `type`, `filter` |
| `GET` | `/api/followers` | 팔로워 목록 JSON API | `sort`, `search`, `from_date`, `to_date` |
| `GET` | `/api/unfollowers` | 언팔 목록 JSON API | `tab` |
| `GET` | `/api/activity` | 활동 검색 JSON API | `username`, `type`, `activity_type` |
| `GET` | `/api/export/csv` | CSV 파일 다운로드 | `type` (followers/unfollowers/activity) |

### 4.1 주요 API 응답 예시

**`GET /api/followers`**
```json
{
  "total": 1024,
  "followers": [
    {
      "username": "john_doe",
      "profile_url": "https://www.instagram.com/john_doe",
      "followed_at": "2024-03-15 14:32:00",
      "timestamp": 1710509520
    }
  ]
}
```

**`GET /api/activity?username=john_doe&type=received`**
```json
{
  "username": "john_doe",
  "total": 12,
  "activities": [
    {
      "type": "like",
      "post_url": "https://www.instagram.com/p/AbCdEfG/",
      "occurred_at": "2024-02-28 09:11:00"
    },
    {
      "type": "comment",
      "post_url": "https://www.instagram.com/p/XyZ12345/",
      "content": "정말 멋진 사진이네요!",
      "occurred_at": "2024-03-10 21:45:00"
    }
  ]
}
```

## 5. 웹 UI 화면 설계

### 5.1 공통 레이아웃
- 상단 내비게이션 바: 로고 / 팔로워 / 언팔 / 활동 검색 메뉴
- 사이드 통계 카드: 전체 팔로워 수, 전체 팔로잉 수, 맞팔 비율
- 반응형 레이아웃 (Bootstrap 5 활용)
- 인스타그램 브랜드 컬러 (`#E1306C` 핑크, `#833AB4` 퍼플) 테마

### 5.2 화면별 구성

| 화면 | 주요 컴포넌트 | 인터랙션 |
|------|-------------|---------|
| 홈 / 업로드 | 드래그&드롭 파일 업로드 영역, 파일 유효성 안내, 업로드 진행 바 | ZIP 또는 JSON 파일 선택 → 자동 파싱 → 대시보드 이동 |
| 팔로워 목록 | 통계 카드, 검색창, 정렬 드롭다운, 날짜 필터, 데이터 테이블, CSV 버튼 | 검색·필터 → AJAX 실시간 갱신 → 클릭 시 인스타그램 프로필 이동 |
| 언팔 목록 | 탭 (맞팔 안 됨 / 언팔당함), 각 탭별 계정 테이블, 마지막 팔로우 날짜 | 탭 전환 → 해당 목록 표시 |
| 활동 검색 | 계정명 검색창, 방향 토글 (받은/보낸), 활동 유형 필터, 타임라인 카드 피드 | 계정명 입력 → 검색 → 날짜순 활동 카드 표시 |

### 5.3 활동 검색 결과 카드 예시

```
┌──────────────────────────────────────────────────────────┐
│ 💬 댓글   @target_username 의 게시물에    2024-03-15 14:32 │
│ ─────────────────────────────────────────────────────── │
│ "정말 멋진 사진이네요! 어디서 찍으셨나요?"                  │
│ 🔗 게시물 바로가기 → https://www.instagram.com/p/AbCdEfG/ │
└──────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────┐
│ ❤️  좋아요  @target_username 의 게시물에   2024-02-28 09:11 │
│ ─────────────────────────────────────────────────────── │
│ 🔗 게시물 바로가기 → https://www.instagram.com/p/XyZ12345/ │
└──────────────────────────────────────────────────────────┘
```

## 6. 개발 구현 계획

### 6.1 기술 스택 상세

| 구분 | 기술 | 용도 |
|------|------|------|
| 백엔드 언어 | Python 3.11+ | 메인 언어 |
| 웹 프레임워크 | Flask 3.x | 라우팅, API, 템플릿 렌더링 |
| 데이터 처리 | Python 표준 `json` 모듈 | JSON 파싱 |
| 데이터 처리 | `pandas` (선택) | CSV 내보내기, 데이터 분석 |
| 프론트엔드 | Bootstrap 5 | 반응형 UI 레이아웃 |
| 프론트엔드 | Vanilla JavaScript (Fetch API) | AJAX 비동기 검색 |
| 날짜 처리 | Python `datetime` | 타임스탬프 변환 |
| 파일 처리 | Python `zipfile` | ZIP 압축 해제 |

### 6.2 단계별 개발 순서

| 단계 | 작업 내용 | 산출물 |
|------|-----------|--------|
| 1단계 | 프로젝트 구조 생성, Flask 기본 설정, 파일 업로드 기능 | `app.py`, 업로드 화면 |
| 2단계 | `parser.py` 구현 — followers/following/likes/comments JSON 파싱 | `parser.py` |
| 3단계 | F-01 팔로워 서비스 + 라우트 + UI 구현 | `follower_service.py`, `followers.html` |
| 4단계 | F-02 언팔 감지 서비스 + 라우트 + UI 구현 | `activity_service.py` (언팔), `unfollowers.html` |
| 5단계 | F-03/F-04 활동 검색 서비스 + 라우트 + UI 구현 | `activity_service.py` (검색), `activity.html` |
| 6단계 | CSV 내보내기, 날짜 필터, 검색 기능 완성 | export 기능 |
| 7단계 | UI 다듬기 (Bootstrap 스타일링, 반응형) | 최종 UI |

### 6.3 설치 및 실행 방법

**`requirements.txt`**
```
flask>=3.0.0
pandas>=2.0.0      # CSV 내보내기용 (선택)
python-dateutil    # 날짜 처리
```

**실행 명령어**
```bash
# 1. 의존성 설치
pip install -r requirements.txt

# 2. Flask 서버 실행
python app.py

# 3. 브라우저에서 접속
# http://localhost:5000
```

## 7. 보안 및 주의사항

### 🔒 개인정보 보호 원칙

- 모든 데이터 처리는 **로컬(사용자 PC)에서만** 수행됩니다. 서버 전송 없음.
- 업로드된 JSON 파일은 분석 후 `data/` 디렉토리에 임시 저장되며, 삭제 버튼으로 즉시 제거 가능합니다.
- 앱은 `localhost`에서만 실행되므로 외부 접근이 차단됩니다.
- 인스타그램 계정 로그인 정보를 수집하거나 저장하지 않습니다.

### ⚠️ 인스타그램 이용 약관

- 이 프로그램은 **공식 데이터 내보내기 파일만 사용**하므로 인스타그램 이용약관을 위반하지 않습니다.
- 비공식 API(스크래핑, 크롤링)는 사용하지 않습니다.
- 타인의 계정 데이터를 무단으로 수집하는 용도로 사용하지 마세요.
